# 기술 스택 - 가이드

## 개요

Sweet Order 웹 판매자 관리 시스템의 실제 구현된 기술 스택과 구체적인 사용 사례, 코드 예시를 포함한 실용적인 가이드입니다.

## 핵심 기술 스택

### 1. Vite

**선택 이유:**

- **빠른 개발 서버**: HMR(Hot Module Replacement)로 즉시 반영
- **간단한 설정**: 복잡한 설정 없이 빠른 개발 환경 구축
- **SEO 불필요**: 관리자 시스템은 SEO가 중요하지 않음
- **SPA 최적화**: 단일 페이지 애플리케이션에 최적화
- **빌드 성능**: 빠른 빌드 속도와 작은 번들 크기

**현재 사용법:**

- **@vitejs/plugin-react**: React 플러그인으로 JSX/TSX 지원
- 경로 별칭(`@/apps/web-seller`)으로 절대 경로 사용
- 개발 서버 포트 3002로 설정
- 빌드 시 소스맵 생성

**향후 계획:**

- 플러그인 확장 (PWA, 번들 분석)
- 빌드 최적화 설정
- 환경별 설정 분리

### 3. Shadcn/ui + Tailwind CSS

**선택 이유:**

- **복사-붙여넣기 방식**: npm 패키지가 아닌 코드 소유권 확보
- **완전한 커스터마이징**: 프로젝트에 맞게 자유롭게 수정 가능
- **Tailwind CSS 기반**: 유틸리티 우선 스타일링으로 빠른 개발
- **Radix UI 기반**: 접근성과 키보드 내비게이션 기본 지원
- **TypeScript 완벽 지원**: 타입 안전성 보장
- **번들 크기 최적화**: 필요한 컴포넌트만 포함

**주요 패키지:**

- **tailwindcss** (v3.4.18): 유틸리티 우선 CSS 프레임워크
- **postcss** (v8.5.6): CSS 변환 도구
- **autoprefixer** (v10.4.22): 벤더 프리픽스 자동 추가
- **@radix-ui/react-select**: 접근성 높은 Select 컴포넌트
- **@radix-ui/react-label**: 폼 Label 컴포넌트
- **@radix-ui/react-tabs**: Tab 인터페이스 컴포넌트
- **@radix-ui/react-slot**: 컴포넌트 합성 유틸리티
- **lucide-react**: 깔끔하고 일관된 아이콘 라이브러리
- **class-variance-authority**: Variant 기반 스타일 관리 (CVA)
- **tailwind-merge**: Tailwind 클래스 충돌 해결
- **clsx**: 조건부 클래스 네이밍
- **tailwindcss-animate**: 애니메이션 유틸리티

**구현된 Shadcn 컴포넌트 (src/common/components/ui/):**

- **button.tsx**: 6개 variant (default, destructive, outline, secondary, ghost, link) + 4개 size
- **input.tsx**: 표준 입력 필드
- **label.tsx**: 폼 라벨 (Radix Label 기반)
- **select.tsx**: 드롭다운 선택 (Radix Select 기반)
- **card.tsx**: 카드 레이아웃 (CardHeader, CardTitle, CardDescription, CardContent, CardFooter)
- **alert.tsx**: 알림 메시지 (Alert, AlertTitle, AlertDescription) + variant 지원
- **badge.tsx**: 뱃지/태그 (4개 variant)
- **tabs.tsx**: 탭 인터페이스 (Tabs, TabsList, TabsTrigger, TabsContent)

**현재 사용법:**

- **globals.css**로 Tailwind 전역 스타일 설정
- **HSL 색상 변수**로 다크모드 지원 준비
- **CVA**로 variant 패턴 기반 컴포넌트 스타일링
- **AdminLayout**으로 관리자 레이아웃 구성
- 모든 폼, 페이지, 레이아웃에서 Shadcn 컴포넌트 활용
- **23개 파일에서 52번** Shadcn 컴포넌트 import 사용 중

**마이그레이션 완료:**

- Material-UI에서 Shadcn/ui로 완전 마이그레이션 완료 (2025-11-28)
- MUI 관련 의존성 완전 제거
- 29개 컴포넌트/페이지 마이그레이션 완료
- lucide-react로 아이콘 통합

### 4. React Router

**선택 이유:**

- **SPA 라우팅**: 클라이언트 사이드 라우팅
- 최신 버전의 성능 개선 및 기능

**현재 사용법:**

- BrowserRouter로 라우팅 설정
- Routes와 Route로 페이지 구성
- ROUTE_CONFIG로 중앙화된 라우트 관리
- Navigate로 리다이렉트 처리

### 5. TanStack Query (v5)

**선택 이유:**

- 서버 상태 관리의 복잡성 해결
- 캐싱, 동기화, 백그라운드 업데이트 자동화
- React와의 완벽한 통합

**현재 사용법:**

- QueryProvider로 전역 설정
- 인증 관련 쿼리 관리
- 자동 캐싱과 리페칭 설정
- 쿼리 무효화를 통한 데이터 동기화

**개발 도구:**

- **@tanstack/react-query-devtools** (v5): 개발 환경에서 쿼리 상태 모니터링

### 6. Zustand (v5)

**선택 이유:**

- 간단하고 직관적인 상태 관리
- Redux 대비 보일러플레이트 감소
- TypeScript와의 우수한 호환성

**현재 사용법:**

- 인증 상태 관리 (로그인/로그아웃, 사용자 정보)
- 전역 알림 상태 관리 (에러, 성공, 경고 메시지)
- 간단한 클라이언트 상태 관리

### 7. Axios

**선택 이유:**

- 인터셉터를 통한 요청/응답 처리
- 자동 JSON 변환과 에러 처리
- 쿠키 기반 인증 지원

**현재 사용법:**

- API 클라이언트 설정 (baseURL, timeout, credentials)
- 인터셉터를 통한 토큰 자동 갱신
- 에러 처리와 재시도 로직
- 쿠키 기반 인증 (서브도메인 통합 로그인)

### 8. react-error-boundary

**선택 이유:**

- React 컴포넌트 트리에서 발생하는 에러를 안전하게 처리
- 사용자에게 친화적인 에러 UI 제공

**현재 사용법:**

- ErrorBoundaryProvider로 전역 에러 처리
- 에러 발생 시 에러 페이지 표시

### 9. react-quill

**선택 이유:**

- 리치 텍스트 에디터
- 관리자 시스템에서 상품 설명 등 콘텐츠 작성에 필요

**현재 사용법:**

- 상품 설명, 공지사항 등 HTML 콘텐츠 작성
