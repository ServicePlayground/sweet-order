# 기술 스택 - 가이드

## 개요

Sweet Order 웹 판매자 관리 시스템의 실제 구현된 기술 스택과 구체적인 사용 사례, 코드 예시를 포함한 실용적인 가이드입니다.

## 핵심 기술 스택

### 1. Vite

**선택 이유:**

- **빠른 개발 서버**: HMR(Hot Module Replacement)로 즉시 반영
- **간단한 설정**: 복잡한 설정 없이 빠른 개발 환경 구축
- **SEO 불필요**: 관리자 시스템은 SEO가 중요하지 않음
- **SPA 최적화**: 단일 페이지 애플리케이션에 최적화
- **빌드 성능**: 빠른 빌드 속도와 작은 번들 크기

**현재 사용법:**

- **@vitejs/plugin-react**: React 플러그인으로 JSX/TSX 지원
- 경로 별칭(`@/apps/web-seller`)으로 절대 경로 사용
- 개발 서버 포트 3002로 설정
- 빌드 시 소스맵 생성

**향후 계획:**

- 플러그인 확장 (PWA, 번들 분석)
- 빌드 최적화 설정
- 환경별 설정 분리

### 2. React (v18) MUI에서 React18 의존

### 3. Material-UI (v5)

**선택 이유:**

- **일관된 디자인**: Material Design 가이드라인
- **빠른 개발 속도**: 빠른 개발 속도와 높은 생산성
- **풍부한 컴포넌트**: 관리자 시스템에 필요한 모든 컴포넌트
- **테마 시스템**: 커스텀 테마 적용 가능
- **TypeScript 지원**: 완벽한 타입 지원

**주요 패키지:**

- **@mui/material** (v5.15.14): 핵심 컴포넌트 라이브러리
- **@mui/icons-material** (v5.15.0): Material Design 아이콘
- **@mui/system** (v5.15.14): 스타일링 시스템
- **@mui/x-data-grid** (v8.15.0): 데이터 그리드 컴포넌트
- **@mui/x-date-pickers** (v8.15.0): 날짜 선택기 컴포넌트
- **@emotion/react** (v11.11.0): CSS-in-JS 엔진
- **@emotion/styled** (v11.11.0): 스타일드 컴포넌트

**현재 사용법:**

- ThemeProvider로 전역 테마 설정
- CssBaseline으로 기본 스타일 리셋
- AdminLayout으로 관리자 레이아웃 구성
- 데이터 그리드, 날짜 선택기 등 관리자 UI 컴포넌트 활용

### 4. React Router

**선택 이유:**

- **SPA 라우팅**: 클라이언트 사이드 라우팅
- 최신 버전의 성능 개선 및 기능

**현재 사용법:**

- BrowserRouter로 라우팅 설정
- Routes와 Route로 페이지 구성
- ROUTE_CONFIG로 중앙화된 라우트 관리
- Navigate로 리다이렉트 처리

### 5. TanStack Query (v5)

**선택 이유:**

- 서버 상태 관리의 복잡성 해결
- 캐싱, 동기화, 백그라운드 업데이트 자동화
- React와의 완벽한 통합

**현재 사용법:**

- QueryProvider로 전역 설정
- 인증 관련 쿼리 관리
- 자동 캐싱과 리페칭 설정
- 쿼리 무효화를 통한 데이터 동기화

**개발 도구:**

- **@tanstack/react-query-devtools** (v5): 개발 환경에서 쿼리 상태 모니터링

### 6. Zustand (v5)

**선택 이유:**

- 간단하고 직관적인 상태 관리
- Redux 대비 보일러플레이트 감소
- TypeScript와의 우수한 호환성

**현재 사용법:**

- 인증 상태 관리 (로그인/로그아웃, 사용자 정보)
- 전역 알림 상태 관리 (에러, 성공, 경고 메시지)
- 간단한 클라이언트 상태 관리

### 7. Axios

**선택 이유:**

- 인터셉터를 통한 요청/응답 처리
- 자동 JSON 변환과 에러 처리
- 쿠키 기반 인증 지원

**현재 사용법:**

- API 클라이언트 설정 (baseURL, timeout, credentials)
- 인터셉터를 통한 토큰 자동 갱신
- 에러 처리와 재시도 로직
- 쿠키 기반 인증 (서브도메인 통합 로그인)

### 8. react-error-boundary

**선택 이유:**

- React 컴포넌트 트리에서 발생하는 에러를 안전하게 처리
- 사용자에게 친화적인 에러 UI 제공

**현재 사용법:**

- ErrorBoundaryProvider로 전역 에러 처리
- 에러 발생 시 에러 페이지 표시

### 9. react-quill

**선택 이유:**

- 리치 텍스트 에디터
- 관리자 시스템에서 상품 설명 등 콘텐츠 작성에 필요

**현재 사용법:**

- 상품 설명, 공지사항 등 HTML 콘텐츠 작성
