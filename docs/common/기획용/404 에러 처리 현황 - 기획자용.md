# 404 에러 처리 현황 - 기획자용

## 개요

web-user, web-seller, backend에서 404 에러가 발생하는 상황과 현재 처리 방식을 정리한 문서입니다.

---

## 1. Backend (NestJS)

### 1.1 발생 상황

- **리소스 404**: 요청한 리소스(스토어, 상품, 주문, 피드 등)가 데이터베이스에 존재하지 않는 경우
- 예시:
  - 스토어 조회 시 스토어가 없을 때
  - 상품 조회 시 상품이 없을 때
  - 주문 조회 시 주문이 없을 때
  - 피드 조회 시 피드가 없을 때
- 사용자에게 표시:
  - 에러 얼럿 표시 (예: "스토어를 찾을 수 없습니다.")

## 2. Web-User (Next.js)

### 2.1 발생 상황

- **페이지 라우트 404**: 존재하지 않는 URL 경로로 접근한 경우
- 사용자에게 표시:
  - 현재 구현되어 있지 않음

## 3. Web-Seller (React Router)

### 3.1 발생 상황

- **페이지 라우트 404**: 존재하지 않는 URL 경로로 접근한 경우
- 사용자에게 표시:
  - 현재 구현되어 있지 않음
